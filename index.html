Wow Body – Fitness Weight Loss App

A web product already generating revenue — but hemorrhaging subscribers. The client needed a mobile experience that could retain users and carve out a distinct position in a crowded weight loss market.

Mobile Design Health & Fitness 0→1 Mobile Retention B2C
Product Designer ↗

The challenge I walked into

Active web product, high churn, undifferentiated in a saturated market.

Process
Discovery → Insights → Core Flows → Features
Key Decisions
  • Discovery Scope — Built user personas, CJM, and core flows. Uncovered two key hypotheses: users want the minimum effective dose of exercise, and progress tracking drives accountability.
  • Onboarding — Designed a 5-step flow to personalize the experience from day one and reduce early drop-off.
  • Dish Page — Built a flexible meal system letting users swap dishes and adjust servings — removing the biggest friction point in long-term diet adherence.
  • Weight Logging — Simplified progress tracking to a single input. Lower friction, higher consistency.
  • Workouts — Curated minimal, goal-specific routines — enough to stay fit, not enough to overwhelm.
Insights
80% Didn't know basic exercises
52% Accessed between 1–3pm
12% Logged their weight

2025

Messaging Module Redesign

A critical messaging module serving 42M+ users — outdated, undocumented, and impossible to maintain. After a security breach exposed the risks, the product needed a full rethink.

Enterprise SaaS B2B Legacy Redesign Security Complex Systems
Product Owner / Designer ↗

The Challenge

The module was built around system constraints, not user needs. Template creation averaged 12 minutes. The editor was rigid. Unsafe HTML imports created active security vulnerabilities. I led the end-to-end redesign — from audit to launch.

Process
Requirements Gathering → Problem Framing → Flow Redesign → Security Coverage → Launch
Key Decisions
  • Requirements Gathering — Conducted stakeholder interviews to align business constraints with user needs, defining clear security and usability targets.
  • Problem Framing — Mapped the legacy workflow, identified three critical failure points: inefficiency, poor usability, and security risk.
  • Flow Redesign — Rebuilt the template creation flow from scratch. Removed redundant steps, collapsed metadata and editing into a single page.
  • Security Coverage — Designed a "Check Code" gate into the creation flow, eliminating harmful script imports at the source.
  • Launch — Delivered a flexible drag-and-drop editor replacing the rigid legacy interface.
Outcome
4× Faster Template creation (12m → 3m)
2 Wks Early Shipped ahead of deadline
AI-Ready Scalable, secure foundation

2023

EHR/EMR Platform Redesign

A leading US healthcare platform used by physicians, nurses, and analysts — but too complex to use without lengthy training. The system needed a complete UX overhaul while staying within a strict component library and serving users across multiple continents.

Enterprise SaaS Healthcare Design System Multi-stream B2B
UX/UI Designer ↗

The Challenge

The platform handled everything from diagnoses to treatment planning for millions of patients — but medical staff were bypassing it entirely, defaulting to paper notes and verbal handoffs. The interface was too complex for the clinical pace. I worked across four product streams: Care Planning, Care Team, Condition Management, and Confidentiality.

Process
Sprints → SME Validation → Test Loops → System Design
Key Decisions
  • Roadmap & Sprint Structure — Work was planned annually and executed in quarterly tactical sprints, each scoped around testable features.
  • SME Validation — Before every test cycle, solutions were reviewed with practicing doctors and nurses — catching real-world friction before it reached prototypes.
  • Prototype & Test Loop — Ran iterative testing in Figma, Sketch, and Axure. Key finding: clinical staff avoided the system in favor of paper — which reframed the core challenge from "complexity" to "speed of use."
  • Component-Based Design — Built and maintained a custom design system on top of two existing component libraries, enabling fast iteration across all streams.
Outcome
< 1 Year Delivered launch-ready product
Scaled Adopted by multiple sub-products
Acquired By a top-tier US tech company
